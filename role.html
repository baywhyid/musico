Kategori ARIA roles

Ada enam kategori utama ARIA roles:

Document structure roles

Widget roles

Landmark roles

Live region roles

Window roles

Abstract roles

Document Structure Roles

Role ini mendefinisikan struktur keseluruhan halaman web. Dengan ini, teknologi bantu bisa memahami hubungan antar bagian halaman dan membantu navigasi.

Namun, sebagian besar role ini jarang dipakai sekarang karena HTML modern sudah punya elemen semantik yang setara. Jadi, sebaiknya gunakan elemen HTML semantik jika tersedia.

Gunakan role hanya jika tidak ada elemen HTML yang setara, seperti:

toolbar

tooltip

feed

math

presentation

none

note

Contoh:

<div role="math" aria-label="x squared + y squared = 3">
  x<sup>2</sup> + y<sup>2</sup> = 3
</div>

Di sini juga digunakan atribut aria-label untuk menjelaskan isi elemen.

Widget Roles

Role ini menentukan fungsi dan perilaku elemen interaktif, seperti scrollbar.

Contoh role:

scrollbar

searchbox

separator (jika bisa difokuskan)

slider

spinbutton

switch

tab

tabpanel

treeitem

Contoh searchbox:

<div class="search-container" role="search">
  <label for="searchbox" class="visually-hidden">Search</label>

  <div
    id="searchbox"
    class="searchbox"
    role="searchbox"
    aria-label="Search the site"
    tabindex="0"
    contenteditable="true">
  </div>

  <button type="button" aria-label="Submit search">Search</button>
</div>

Beberapa role ini punya elemen HTML yang setara. Jika ada, utamakan elemen HTML tersebut. Misalnya, lebih baik pakai <button> daripada <div role="button">.

Landmark Roles

Role ini digunakan untuk menandai bagian utama halaman. Screen reader menggunakannya untuk navigasi cepat.

Contoh:

banner

complementary

contentinfo

form

main

navigation

region

search

Semua ini punya padanan HTML:

header

footer

aside

form

main

nav

section

Jika kamu sudah menggunakan elemen HTML yang tepat, tidak perlu menambahkan role lagi.

Contoh:

<div role="banner" class="site-banner">
  <h1>Accessible Web Design</h1>
  <nav>
    <ul>
      <li><a href="#">Home</a></li>
      <li><a href="#">Articles</a></li>
      <li><a href="#">About</a></li>
      <li><a href="#">Contact</a></li>
    </ul>
  </nav>
</div>

Live Region Roles

Role ini digunakan untuk elemen yang isinya berubah secara dinamis. Screen reader akan memberitahu pengguna jika ada perubahan.

Contoh:

alert

log

marquee

status

timer

Contoh:

<div class="status-demo">
  <button id="update-status-btn">Check Status</button>
  <div id="status-msg" role="status">
    No updates yet.
  </div>
</div>

Saat isi berubah, screen reader akan mengumumkannya.

Window Roles

Role ini digunakan untuk sub-window seperti dialog atau popup.

Contoh:

alertdialog

dialog

Namun sekarang lebih disarankan menggunakan elemen HTML <dialog> daripada membuat sendiri dengan role.

Contoh:

<div id="custom-dialog" role="dialog" aria-modal="true" aria-labelledby="dialog-title">

Abstract Roles

Role ini hanya digunakan secara internal oleh browser untuk mengatur struktur dokumen.

👉 Developer tidak perlu menggunakannya.

Kesimpulan

Dengan ARIA roles, kamu bisa membuat website dan aplikasi web yang lebih aksesibel.

ARIA memberikan informasi tambahan tentang fungsi dan tujuan elemen HTML, sehingga screen reader dan teknologi bantu bisa membantu pengguna memahami dan menggunakan halaman dengan lebih baik.

👉 Tujuannya: memastikan semua orang bisa mendapatkan pengalaman yang baik saat menggunakan web.
